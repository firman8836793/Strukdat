Materi Stack

Definisi Stack

Sebuah kumpulan item berurutan di mana penambahan item baru dan penghapusan item yang sudah ada selalu terjadi pada ujung yang sama. Ujung ini umumnya disebut sebagai “puncak” (top). Ujung yang berlawanan dengan puncak dikenal sebagai “dasar” (base). Prinsip pengurutan ini terkadang disebut LIFO (Last-In, First-Out / Terakhir Masuk, Pertama Keluar) atau FILO (First-In, Last-Out / Pertama Masuk, Terakhir Keluar).

Prinsip LIFO (Last In First Out)

Prinsip LIFO berarti elemen yang paling akhir ditambahkan ke dalam stack adalah elemen pertama yang akan dihapus atau diambil.

  • Elemen baru selalu ditambahkan (didorong/pushed) ke posisi paling atas.
  • Penghapusan (pop) juga hanya terjadi dari posisi atas.
  • Aturan ini memastikan urutan yang ketat : terakhir masuk -> pertama keluar

Operasi Basic Pada Stack

  • Stack() : membuat sebuah stack (tumpukan) baru yang kosong. Operasi ini tidak membutuhkan parameter dan mengembalikan sebuah stack yang kosong.
  • Push(): Menambahkan elemen baru ke atas stack.
  • Pop(): Menghapus dan mengembalikan elemen teratas dari stack.
  • Peek(): Mengembalikan (melihat) elemen teratas (terakhir) pada stack.
  • isEmpty(): Memeriksa apakah stack tersebut kosong.
  • Size(): Mengetahui jumlah elemen di dalam stack.

Ekspresi Infix, Prefix , dan Postfix

Definisi

Dalam dunia struktur data dan pemrograman, infix, prefix, dan postfix adalah tiga cara berbeda untuk menuliskan ekspresi matematika. Perbedaannya terletak pada letak operator (seperti +, - , x , / terhadap operan (angka atau variabel). Maka Operasi perhitungannya mengikuti level dari precedence tersebut. A + B * C + D = ((A + (B * C)) + D) 🡪 operator * lebih didahulukan daripada operator *, dan operator + yang pertama juga didahulukan daripada yang kedua (kiri ke kanan) Jadi prinsip Precedence level adalah : Highest Precedence and Left to Right

1.Infix (Notasi Umum)

Infix adalah notasi yang paling sering kita gunakan dalam kehidupan sehari-hari. Pada notasi ini, operator diletakkan di tengah antara dua operan. Format: Operan1 [Operator] Operan2 Karakteristik: Memerlukan tanda kurung () dan aturan derajat operator (pangkat, kali/bagi, tambah/kurang) untuk menentukan urutan pengerjaan. Contoh: A + B atau (A + B) * C

2. Prefix (Polish Notation)

Prefix adalah notasi di mana operator diletakkan di depan sebelum operan. Notasi ini diciptakan oleh Jan Łukasiewicz untuk menghilangkan kebutuhan akan tanda kurung. Format: [Operator] Operan1 Operan2Karakteristik: Komputer membaca dari kanan ke kiri untuk mengevaluasi ekspresi ini. Tidak butuh tanda kurung karena urutan operasinya sudah tetap.Contoh: + A B (artinya sama dengan A+B)

3. Postfix (Reverse Polish Notation / RPN)

Postfix adalah kebalikan dari prefix, di mana operator diletakkan di belakang setelah operan. Ini adalah format yang paling disukai oleh mesin/komputer (menggunakan struktur data Stack) karena sangat efisien untuk diproses. Format: Operan1 Operan2 [Operator]Karakteristik: Komputer membaca dari kiri ke kanan. Tidak membutuhkan tanda kurung atau aturan prioritas operator yang rumit saat diproses oleh mesin.Contoh: A B + (artinya sama dengan A+B)

Algoritma Evaluasi & Konversi

Infix ke Postfix:

  • Scan ekspresi dari kiri ke kanan.
  • Jika operan, masukkan ke daftar output.
  • Jika operator, bandingkan prioritasnya dengan yang ada di stack. Hapus operator di stack yang memiliki prioritas lebih tinggi/sama dan masukkan ke output, lalu masukkan operator baru ke stack.

Infix ke Prefix:

  • Balikkan ekspresi infix (tanda kurung juga dibalik).
  • Ubah ekspresi yang sudah dibalik tersebut menjadi postfix.
  • Balikkan kembali hasil postfix tersebut untuk mendapatkan hasil prefix.

Evaluasi Postfix:

  • Scan dari kiri ke kanan.
  • Jika operan, masukkan ke stack.
  • Jika operator, ambil (pop) dua operan dari stack, hitung hasilnya, dan masukkan kembali ke stack.

def infixToPrefix(s):
    st = []
    result = []

    # scan from right to left
    for c in reversed(s):
        if c.isalnum():
            result.append(c)
        elif c == ')':
            st.append(c)
        elif c == '(':
            while st and st[-1] != ')':
                result.append(st.pop())
            if st:
                st.pop()  # remove ')'
        elif isOperator(c):
            while (st and isOperator(st[-1]) and
                  (precedence(st[-1]) > precedence(c) or
                  (precedence(st[-1]) == precedence(c) and isRightAssociative(c)))):
                result.append(st.pop())
            st.append(c)

    # pop remaining operators
    while st:
        result.append(st.pop())

    # reverse at the end to get correct prefix
    return ''.join(reversed(result))

DEFINISI QUEUE

Sebuah kumpulan item berurutan di mana penambahan item baru terjadi pada satu ujung, yang disebut “belakang” (rear), dan penghapusan item yang sudah ada terjadi pada ujung lainnya, yang umumnya disebut “depan” (front). Prinsip pengurutan ini terkadang disebut FIFO, first-in first-out (pertama masuk, pertama keluar). Ini juga dikenal sebagai prinsip “pertama datang, pertama dilayani” (first-come first-served).

Oprasi dalam Queue

  • Queue(): Membuat sebuah queue (antrean) baru yang kosong. Operasi ini tidak membutuhkan parameter dan akan mengembalikan sebuah antrean kosong.
  • enqueue(item): Menambahkan sebuah item baru ke bagian belakang (rear) antrean. Operasi ini membutuhkan parameter berupa item yang akan dimasukkan dan tidak mengembalikan nilai apa pun.
  • dequeue(): Menghapus item terdepan (front) dari antrean. Operasi ini tidak membutuhkan parameter dan akan mengembalikan item yang dihapus tersebut. Struktur antrean akan berubah (termodifikasi).
  • isEmpty(): Menguji atau memeriksa apakah antrean dalam keadaan kosong. Operasi ini tidak membutuhkan parameter dan akan mengembalikan nilai boolean (True atau False).
  • size(): Mengembalikan jumlah item yang ada di dalam antrean. Operasi ini tidak membutuhkan parameter dan akan mengembalikan nilai bilangan bulat (integer).

Definisi Deques

Deques juga dikenal sebagai double-ended queue (antrean dua ujung), adalah kumpulan item berurutan yang mirip dengan queue (antrean). Struktur ini memiliki dua ujung, yaitu depan (front) dan belakang (rear), dan item-item tersebut tetap berada pada posisinya di dalam kumpulan tersebut.

Oprasi dalam Deques

  • Deque(): Membuat sebuah deque baru yang kosong. Operasi ini tidak membutuhkan parameter dan akan mengembalikan sebuah deque kosong.
  • addFront(item): Menambahkan sebuah item baru ke bagian depan (front) dari deque. Operasi ini membutuhkan parameter berupa item yang akan ditambahkan dan tidak mengembalikan nilai apa pun.
  • addRear(item): Menambahkan sebuah item baru ke bagian belakang (rear) dari deque. Operasi ini membutuhkan parameter berupa item yang akan ditambahkan dan tidak mengembalikan nilai apa pun.
  • removeFront(): Menghapus item terdepan dari deque. Operasi ini tidak membutuhkan parameter dan akan mengembalikan item yang dihapus tersebut. Struktur deque akan berubah (termodifikasi).
  • removeRear(): Menghapus item paling belakang dari deque. Operasi ini tidak membutuhkan parameter dan akan mengembalikan item yang dihapus tersebut. Struktur deque akan berubah (termodifikasi).
  • isEmpty(): Menguji atau memeriksa apakah deque dalam keadaan kosong. Operasi ini tidak membutuhkan parameter dan akan mengembalikan nilai boolean (True atau False).
  • size(): Mengembalikan jumlah item yang ada di dalam deque. Operasi ini tidak membutuhkan parameter dan akan mengembalikan nilai bilangan bulat (integer).

2. Unordered Linked List

Linked list merupakan kumpulan dari node-node yang terhubung satu sama lain. Untuk mengaksesnya, diperlukan pointer head yang menunjukkan keberadaan node pertama.

Operasi pada Unordered List:

Materi Hashing

Definisi :¶

1. Hashing¶

Hashing adalah algoritma pencarian di mana semua data terletak tepat berada ditempatnya masing-masing, sehingga pencarian dilakukan hanya dengan satu kali proses perbandingan saja. Di dalam algoritma hashing ini terdapat beberapa istilah dasar:

  • Hash Table: Sebuah tempat penyimpanan data, yang dibuat sedemikian rupa, sehingga dapat memudahkan pencarian. Tipe data list di python dapat digunakan untuk merepresentasikan hash table.
  • Slot: Posisi (indeks) yang terdapat pada hash table sebagai tempat penyimpanan setiap data. Nilai slot adalah integer mulai dari nol sampai dengan ukuran dari hash table.
  • Hash Function: Suatu fungsi yang memetakan antara data dengan slot di dalam hash table.

Cara Kerja:¶

  • Data di dalam list disusun berdasarkan nilai hash, dan pencarian data dilakukan berdasarkan nilai hash dari hash function.
  • Pencarian data dilakukan hanya dengan cara menghitung function hash berdasarkan data tersebut.
  • Nilai balik dari perhitungan tersebut merupakan index atau slot dari data di dalam list.
  • Kemudian dilakukan pengecekan keberadaan data tersebut hanya dengan satu kali perbandingan saja.
  • Jika terdapat data dengan nilai hash yang sama sehingga mengacu pada nomor slot yang sama, maka akan terjadi masalah yang disebut collusion.
  • Collusion ini dapat ditangani dengan konsep open addressing (mencari slot kosong) seperti Linear Probing atau Quadratic Probing, maupun dengan metode Chaining di mana satu slot dapat ditempati oleh beberapa data sekaligus.

Fungsi Hash:¶

Hash function memiliki parameter nilai data dan mengembalikan nilai integer (nilai hash) yang merupakan representasi dari nomor slot. Berikut adalah beberapa metode fungsi hash:

  • Remainder Function: Fungsi hash sederhana di mana nilai baliknya berupa modulus dari data tersebut dengan sebuah angka (misalnya ukuran dari tabel). Contoh: terdapat data bernilai 54 dengan ukuran tabel 11, maka nilai hash = 54 % 11, yaitu 4.
  • Mid-Square Method: Ini adalah contoh perfect hash function yang bertujuan meminimalkan terjadinya collusion. Metodenya adalah mengkuadratkan nilai data, kemudian mengambil bagian tertentu (tengah) dari data tersebut, lalu dimodulus dengan ukuran tabel. Contoh: data 35 dikuadratkan menjadi 1225, diambil nilai tengahnya yaitu 22, lalu 22 % 11, hasilnya adalah slot 0.
  • Fungsi Hash untuk String: Hash function juga dapat digunakan untuk tipe data string dengan mengambil jumlah nilai ASCII karakter-karakter penyusun data tersebut menggunakan perintah ord. Untuk menghindari collusion pada kata anagram (contoh 'dia' dan 'adi' yang nilai ASCII totalnya sama), dapat digunakan bobot yang menunjukkan posisi masing-masing karakter untuk dikalikan dengan nilai ASCII-nya.

Materi Tree

Definisi :

1. Tree

Tree merupakan struktur data yang berbentuk non-linear, sangat berbeda dengan struktur data linear seperti stack, queue, ataupun deque. Bentuk tree di dalam computer science memiliki kemiripan dengan pohon di ekosistem biologi karena sama-sama memiliki akar (root), cabang (branches), dan daun (leaves). Bedanya, akar pada tree struktur data terletak di bagian paling atas, sedangkan cabang dan daunnya menjuntai ke bawah.

Istilah-istilah Dasar pada Tree:

  • Node: Bagian dari tree yang menyimpan informasi, nilai, atau key.
  • Edge: Garis yang menghubungkan antara satu node dengan node lainnya. Dalam sebuah node hanya boleh ada satu edge yang masuk, namun bisa ada satu atau banyak edge yang keluar.
  • Root: Node yang posisinya paling atas. Root tidak memiliki edge yang masuk, tapi memiliki satu atau lebih edge yang keluar darinya.
  • Path: Rute atau urutan node dari root menuju ke sebuah node tujuan.
  • Parent & Children: Parent adalah node yang memiliki edge keluar yang mengarah ke node lain. Node yang dituju oleh edge tersebut disebut sebagai Children.
  • Siblings: Node-node (children) yang berasal dari parent yang sama.
  • Subtree: Kumpulan dari node dan edge yang terdiri dari sebuah parent dan semua node yang berada di bawahnya.
  • Leaf Node (Daun): Node yang terletak paling ujung karena tidak memiliki edge yang keluar (tidak punya children).
  • Level & Height: Level adalah tingkatan dari node (Root selalu dihitung sebagai level ke-0, makin ke bawah level makin besar). Height adalah tinggi keseluruhan tree yang didapat dari nilai level maksimumnya.

Definisi :

2. Binary Tree

Binary tree merupakan sebuah tree spesifik di mana setiap parent di dalamnya hanya diizinkan untuk memiliki maksimal dua buah node children, atau satu node child saja.

Karakteristik & Operasi Binary Tree:

  • Representasi Class: Karena di dalam binary tree setiap node harus memiliki informasi mengenai nilai node serta penunjuk/edge ke anak kiri dan kanannya, struktur ini sangat ideal diimplementasikan menggunakan tipe data class.
  • Properti Utama: Class Binary Tree umumnya memiliki tiga properti: key untuk menyimpan nilai, leftChild untuk menunjuk ke anak sebelah kiri, dan rightChild untuk menunjuk ke anak sebelah kanan.
  • Penyisipan Node (Insert): Terdapat method insertLeft dan insertRight. Jika posisi child yang akan diisi masih kosong (None), node baru bisa langsung dimasukkan. Namun, jika posisi child tersebut sudah terisi, maka node baru harus disisipkan di antara parent dan child lama tersebut agar rantai data di bawahnya tidak hilang atau terputus 1438, 1461, 1464, 1492, 1496].

        return self.leftChild

Selection Sort

Definisi :

1. Selection Sort

Selection Sort adalah algoritma pengurutan sederhana yang membagi daftar menjadi dua bagian: bagian yang sudah diurutkan dan bagian yang belum diurutkan. Algoritma ini bekerja dengan cara mencari elemen terkecil secara berulang dari bagian yang belum diurutkan, lalu memindahkan dan menukarnya ke posisi paling depan dari bagian tersebut.

Cara Kerja:

  • Mengasumsikan elemen pertama pada bagian yang belum diurutkan sebagai nilai terkecil sementara (minimum).
  • Membandingkan nilai minimum sementara tersebut dengan elemen-elemen berikutnya dalam sisa daftar.
  • Jika ditemukan elemen yang nilainya lebih kecil (Ist[j]<(minimum)), maka elemen tersebut diperbarui menjadi nilai minimum yang baru.
  • Setelah selesai memeriksa hingga akhir daftar, tukar nilai minimum yang ditemukan dengan elemen pertama dari bagian yang belum diurutkan.
  • Geser batas daftar yang belum diurutkan satu posisi ke kanan, lalu ulangi proses ini hingga seluruh elemen selesai diurutkan.

Definisi :

1. Insertion Sort

Insertion Sort adalah algoritma pengurutan sederhana yang membangun daftar yang diurutkan secara bertahap, elemen demi elemen. Algoritma ini bekerja dengan cara mengambil satu elemen dari daftar yang belum diurutkan dan menyisipkannya ke posisi yang benar di dalam bagian daftar yang sudah diurutkan.

Cara Kerja:

  • Memulai proses dari elemen kedua (indeks ke-1), karena elemen pertama (indeks ke-0) diasumsikan sudah terurut.
  • Menyimpan elemen saat ini sebagai variabel kunci (key) dan membandingkannya dengan elemen-elemen sebelumnya.
  • Jika elemen sebelumnya lebih besar dari kunci (Ist[j]>key), maka elemen tersebut digeser satu posisi ke kanan (Ist[j+1]=Ist[j]).
  • Proses pergeseran ini terus dilakukan ke belakang hingga menemukan elemen yang lebih kecil atau sama dengan kunci, lalu key disisipkan ke posisi yang tepat. Proses ini diulang hingga seluruh daftar selesai diurutkan.

Langkah berikutnya adalah menyisipkan key pada kumpulan data terurut sebelumnya, sehingga data saat ini menjadi [2.5, 7, 5, 10, 1]. Akhir dari iterasi pertama adalah, data pada indeks ke-0 dan indeks ke-1, sudah dalam keadaan terurut

Iterasi Kedua. Pada iterasi kedua menjadikan data pada indeks ke-2, yaitu 5, menjadi key. Sehingga pada iterasi kedua ini, berusaha menyispikan key pada kumpulan data terurut sebelumnya, yaitu [2.5, 7], dengan cara membandingkan satu persatu. Dapat dilihat bahwa data indeks ke-0, sampai dengan indeks ke-2, sudah dalam keadaan terurut

Iterasi Ketiga. Key pada iterasi ketiga adalah data pada indeks ke-3, yaitu 10. Tujuan akhir dari iterasi ketiga adalah mencari lokasi yang tepat untuk data 10 diantara data yang sudah urut sebelumnya, yaitu [2.5, 5, 7]. Counting Sort

Definisi :

1. Counting Sort

Counting Sort adalah algoritma pengurutan yang sangat efisien karena tidak membandingkan elemen satu sama lain. Algoritma ini bekerja dengan cara menghitung jumlah kemunculan (frekuensi) dari setiap nilai unik di dalam daftar, lalu menggunakan informasi tersebut untuk menempatkan elemen tepat pada posisi akhirnya.

Cara Kerja:

  • Mencari nilai tertinggi (maksimum) di dalam daftar untuk menentukan rentang angka.
  • Membuat daftar sementara (count array) untuk menghitung seberapa sering setiap angka muncul.
  • Menjumlahkan nilai di count array secara kumulatif. Nilai kumulatif ini akan menjadi penunjuk indeks pasti di mana angka tersebut harus diletakkan.
  • Menelusuri data awal dari belakang ke depan, lalu menempatkan setiap angka ke daftar hasil (output array) berdasarkan indeks kumulatifnya.
  • Setiap kali angka ditempatkan, nilai kumulatifnya dikurangi satu.

CODE


# Fungsi Counting Sort
def counting_sort(arr):
    # Jika list kosong, langsung kembalikan
    if len(arr) == 0:
        return arr

    # 1. Cari nilai maksimum untuk menentukan ukuran array penghitung
    max_val = max(arr)

    # 2. Siapkan array penghitung (count) dan array hasil (output) dengan nilai awal 0
    count = [0] * (max_val + 1)
    output = [0] * len(arr)

    # 3. Hitung jumlah kemunculan setiap angka
    for num in arr:
        count[num] += 1

    # 4. Ubah array penghitung menjadi jumlah kumulatif
    for i in range(1, len(count)):
        count[i] += count[i - 1]

    # 5. Tempatkan angka ke posisi yang benar di array hasil
    # (Ditelusuri dari belakang agar stabil)
    for i in range(len(arr) - 1, -1, -1):
        num = arr[i]
        output[count[num] - 1] = num
        count[num] -= 1

    return output

Merge Sort

Definisi :

1. Merge Sort

Merge Sort merupakan algoritma pengurutan dengan konsep rekursif yang terdiri dari dua tahapan utama, yaitu Splitting (divide) dan Merging. Algoritma ini didasarkan pada prinsip bahwa jika hanya terdapat satu elemen dalam sebuah list data, maka data tersebut otomatis sudah dalam keadaan terurut.

Cara Kerja:

  • Tahap Divide (Splitting): Merupakan proses Top-Down di mana list data dibagi menjadi dua buah bagian secara terus menerus.
  • Pembagian data ini dilakukan sampai hanya tersisa satu elemen saja didalam list.
  • Tahap Merging: Merupakan tahapan Bottom-up process yang menggabungkan data-data yang sudah terbagi pada tahap divide.
  • Proses merge ini melibatkan proses membandingkan data antara list yang akan digabung.
  • Jika data lebih kecil, maka data tersebut akan diletakkan pada indeks yang lebih rendah.
  • Proses ini akan dilakukan terus menerus, sehingga tidak ada lagi list yang akan digabungkan dan data pada list akan terurut sempurna.

Quick Sort

Definisi :

1. Quick Sort

Quick Sort adalah algoritma pengurutan yang tahapannya terdiri dari proses divide dan combine. Berbeda dengan algoritma merge sort yang membagi data menjadi dua bagian tanpa syarat, pembagian data pada quick sort harus memenuhi ketentuan berdasarkan nilai pivot.

Cara Kerja:

  • Tahap Divide: Membagi data menjadi dua buah bagian berdasarkan nilai pivot.
  • Data yang berada di bagian sebelah kiri (bagian pertama) haruslah data-data yang memiliki nilai lebih kecil dari nilai pivot, meskipun urutannya masih acak.
  • Data yang berada di sebelah kanan (bagian kedua) adalah data-data yang memiliki nilai lebih besar dari nilai pivot.
  • Pada awal partisi, nilai pivot umumnya ditentukan dari data yang terdapat pada indeks awal dari sekumpulan data tersebut.
  • Dilakukan penelusuran data; jika ditemukan data yang nilainya kurang dari pivot, maka data tersebut ditukar posisinya dengan data di sebelah pivot.
  • Pada akhir partisi, pivot diletakkan di posisi indeks sebenarnya seolah-olah data tersebut sudah terurut.
  • Tahap Combine: Tahapan ini pada dasarnya hanya memerintahkan proses partisi secara berulang kali pada sub-bagian data.
  • Bagian data sebelum nilai pivot dan setelah nilai pivot akan dipartisi lagi menjadi dua bagian secara terus-menerus.
  • Proses ini berlanjut sampai tidak ada lagi data yang bisa dipartisi, sehingga hasil akhirnya seluruh data akan terurut.

Divide

Fungsi ini bertugas murni untuk menentukan pivot dan membagi data ke sisi kiri (lebih kecil) dan sisi kanan (lebih besar). 


# TAHAP 1: DIVIDE (Fungsi Partisi)
def partition(A, p, q):
    x = A[p]  # Menentukan pivot dari elemen pertama (indeks p)
    i = p     # Menyimpan batas elemen yang lebih kecil dari pivot

    # Menelusuri sisa data dari p+1 hingga q
    for j in range(p + 1, q + 1):
        # Jika elemen saat ini lebih kecil atau sama dengan pivot
        if A[j] <= x:
            i = i + 1
            # Tukar elemen agar berpindah ke sisi kiri
            A[i], A[j] = A[j], A[i]

    # Tukar posisi pivot ke tengah-tengah batas
    A[p], A[i] = A[i], A[p]
    print(A)
    # Kembalikan indeks posisi pivot yang baru
    return i

def Quicksort(A, p, q):
    # Base case: Lakukan selama indeks awal (p) lebih kecil dari indeks akhir (q)
    if p < q:
        # Panggil tahap DIVIDE untuk memecah data dan mendapatkan indeks pivot (r)

        Quicksort(A, p, r - 1)

        # Panggil tahap COMBINE untuk mengurutkan bagian kanan pivot
        Quicksort(A, r + 1, q)

1. Unordered List Sequential Search¶

Unordered List Sequential Search adalah algoritma pencarian paling dasar yang memeriksa setiap elemen dalam daftar yang tidak terurut secara berurutan satu per satu. Karena datanya acak, algoritma ini harus memeriksa hingga elemen terakhir jika target tidak ditemukan.

Cara Kerja:¶

  • Memulai pencarian dari elemen pertama (indeks ke-0) di dalam daftar.
  • Membandingkan elemen yang sedang diperiksa saat ini dengan nilai target yang dicari (terget)
  • Jika elemen saat ini sama dengan target (Ist[i]==target), maka pencarian berhasil dan indeks dikembalikan.
  • Jika tidak sama, lanjut ke elemen berikutnya.
  • Proses ini terus dilakukan hingga target ditemukan atau pencarian mencapai batas akhir daftar (jika target tidak ada, semua elemen wajib diperiksa).

# 1. Unordered List Sequential Search (Pencarian Tidak Terurut)
def seqSearch(listData, data):

2. Ordered List Sequential Search¶

Ordered List Sequential Search adalah algoritma pencarian sekuensial yang diterapkan pada daftar yang sudah terurut (misalnya, dari kecil ke besar). Keuntungannya adalah pencarian dapat dihentikan lebih awal jika algoritma menemukan elemen yang nilainya lebih besar dari target, karena elemen di sebelahnya pasti akan lebih besar lagi.

Cara Kerja:¶

  • Memulai pencarian dari elemen pertama (indeks ke-0) di dalam daftar.
  • Membandingkan elemen yang sedang diperiksa dengan nilai target (target)
  • Jika elemen saat ini sama dengan target (Ist[i]==target), pencarian berhasil dan indeks dikembalikan.
  • Jika elemen saat ini lebih besar dari target (Ist[i]>target), maka pencarian langsung dihentikan karena tidak mungkin target berada di sisa daftar tersebut.
  • Jika elemen saat ini masih lebih kecil dari target, lanjut ke elemen berikutnya hingga mencapai akhir daftar.

def ordered_sequential_search(arr, target):
    # Telusuri setiap elemen dari awal
    for i in range(len(arr)):
        # Jika target ditemukan, kembalikan indeksnya
        if arr[i] == target:
            return i
        # BERHENTI LEBIH AWAL: Jika elemen saat ini lebih besar dari target
        elif arr[i] > target:
            print(f"Pencarian dihentikan lebih awal di indeks {i} karena {arr[i]} > {target}")
            break # Hentikan loop karena data setelahnya pasti lebih besar

    # Jika tidak ditemukan (atau loop dihentikan lebih awal), kembalikan -1
    return -1

Rekursif

Definisi :

1. Rekursif

Rekursif adalah teknik pemrograman atau pendekatan algoritma di mana sebuah fungsi memanggil dirinya sendiri secara berulang untuk menyelesaikan suatu masalah. Teknik ini bekerja dengan cara memecah masalah yang kompleks menjadi sub-masalah yang lebih kecil dan serupa hingga mencapai kondisi paling sederhana yang dapat langsung diselesaikan.

Cara Kerja Umum:

  • Menentukan Base Case (Kasus Dasar): Ini adalah kondisi berhenti atau titik akhir di mana fungsi tidak lagi memanggil dirinya sendiri, sehingga mencegah terjadinya perulangan tak terbatas (infinite loop).
  • Menentukan Recursive Case (Kasus Rekursif): Bagian di mana fungsi memanggil dirinya sendiri dengan argumen atau nilai masukan yang dimodifikasi, sehingga secara bertahap kondisinya semakin mendekati Base Case.
  • Setiap kali fungsi memanggil dirinya sendiri, status eksekusi saat itu akan ditunda dan disimpan di dalam tumpukan memori komputer (call stack).
  • Begitu Base Case terpenuhi, fungsi berhenti memanggil dirinya sendiri dan mulai mengembalikan hasil eksekusi dari tumpukan memori secara mundur hingga mencapai pemanggilan fungsi yang pertama kali.

Implementasi 1: Faktorial

  • Faktorial adalah operasi matematika yang mengalikan sebuah bilangan bulat positif dengan semua bilangan bulat positif di bawahnya (contoh: 5!=5x4x3x2x1).
  • Base Case: Jika nilai n=0 atau n=1, maka fungsi mengembalikan nilai 1.
  • Recursive Case: Jika n>1, maka fungsi mengembalikan nilai N x faktorial(n-1).
  • Fungsi akan terus memanggil dirinya sendiri untuk mengalikan angka saat ini dengan angka yang lebih kecil hingga mencapai angka 1, lalu hasil akhirnya dihitung secara mundur.

CODE


def faktorial(n):
    # Base Case: jika n adalah 0 atau 1
    if n == 0 or n == 1:
        return 1
    # Recursive Case: n dikali faktorial dari n-1
    else:
        return n * faktorial(n-1)

Implementasi 2: Menara Hanoi (Tower of Hanoi)

  • Menara Hanoi adalah permainan teka-teki matematis dengan tujuan memindahkan tumpukan cakram dari tiang asal ke tiang tujuan, dengan syarat cakram yang lebih besar tidak boleh diletakkan di atas cakram yang lebih kecil.
  • Base Case: Jika hanya ada 1 cakram (n=1), langsung pindahkan cakram tersebut dari tiang asal ke tiang tujuan.
  • Recursive Case: Untuk memindahkan n cakram, logika pemecahannya dibagi menjadi tiga pemanggilan berulang: 1. Pindahkan n-1 cakram bagian atas dari tiang asal ke tiang bantuan (menggunakan tiang tujuan sebagai perantara). 2. Pindahkan 1 cakram terbesar (yang tersisa di paling bawah) dari tiang asal langsung ke tiang tujuan. 3. Pindahkan kembali n-1 cakram dari tiang bantuan ke tiang tujuan (menggunakan tiang asal sebagai perantara).

CODE


def menara_hanoi(n, asal, tujuan, bantuan):
    # Base Case: jika hanya ada 1 cakram, langsung pindahkan
    if n == 1:
        print(f"Pindahkan cakram 1 dari tiang {asal} ke tiang {tujuan}")
        return

    # Recursive Case:
    # 1. Pindahkan n-1 cakram dari asal ke bantuan
    menara_hanoi(n-1, asal, bantuan, tujuan)

    # 2. Pindahkan cakram terbesar (n) dari asal ke tujuan
    print(f"Pindahkan cakram {n} dari tiang {asal} ke tiang {tujuan}")

    # 3. Pindahkan n-1 cakram dari bantuan ke tujuan
    menara_hanoi(n-1, bantuan, tujuan, asal)

Parsing Tree

EDefinisi :

1. Aplikasi Binary Tree: Parsing Ekspresi Matematika

Struktur data Binary Tree dapat diimplementasikan untuk berbagai aplikasi, salah satunya yang paling umum adalah penyelesaian persamaan matematika.

  • Hirarki Operasi: Penyelesaian matematika sangat bergantung pada derajat operator (operator precedence) dan tanda kurung. Operasi di dalam kurung harus diselesaikan terlebih dahulu meskipun derajat operatornya lebih rendah. Binary tree dapat memodelkan hirarki ini di mana left subtree dan right subtree akan dieksekusi terlebih dahulu sebelum dikembalikan ke parent-nya.
  • Tahapan Utama: Proses ini terbagi menjadi dua tahap utama: 1) Pembentukan Parse Tree, dan 2) Evaluasi Persamaan Matematika.

Aturan Pembentukan Parse Tree:

Persamaan matematika pertama-tama harus dipecah menjadi kumpulan token, yaitu kurung buka, kurung tutup, operand (angka), dan operator. Algoritma penyusunannya adalah sebagai berikut:

  • Jika token adalah ( : Tambahkan sebuah node baru kosong sebagai left child dari current node, kemudian pindahkan posisi current node ke node left child yang baru dibuat tersebut.
  • Jika token adalah Operator (+, -, *, /) : Set nilai key dari current node dengan simbol operator tersebut. Setelah itu, buat node baru sebagai right child, lalu pindahkan current node ke posisi right child tersebut.
  • Jika token adalah Operand (Angka) : Ubah nilai key dari current node menjadi angka tersebut. Setelah angkanya masuk, kembalikan posisi current node ke parent-nya.
  • Jika token adalah ) : Proses untuk cabang tersebut selesai, sehingga posisi current node cukup dikembalikan ke parent-nya.

Peran Stack:

Untuk menjalankan algoritma di atas, program butuh kembali ke parent node. Karena struktur class Binary Tree biasa tidak menyimpan alamat parent, maka dibutuhkan struktur data tambahan yaitu Stack. Setiap kali turun ke child, parent akan di-push ke Stack, dan saat perlu kembali ke atas, parent akan di-pop dari Stack.

def buildParseTree(mathExp):
    # Memecah persamaan menjadi list token berdasarkan spasi
    tokenList = mathExp.split()
    parentStack = stack()
    expTree = BinaryTree('')

    push(parentStack, expTree)
    currentTree = expTree

    for i in tokenList:
        if i == '(':
            currentTree.insertLeft('')
            push(parentStack, currentTree)
            currentTree = currentTree.getLeftChild()

        elif i not in ['+', '-', '*', '/', ')']:
            # Token berupa Operand (Angka)
            currentTree.setRootVal(int(i))
            parent = pop(parentStack)
            currentTree = parent

        elif i in ['+', '-', '*', '/']:
            # Token berupa Operator
            currentTree.setRootVal(i)
            currentTree.insertRight('')
            push(parentStack, currentTree)
            currentTree = currentTree.getRightChild()

        elif i == ')':
            currentTree = pop(parentStack)

        else:
            raise ValueError

    return expTree

2. Evaluasi Persamaan Matematika

Setelah parse tree berhasil dibentuk, tahap selanjutnya adalah menyelesaikan (mengevaluasi) nilai dari persamaan matematika tersebut.

  • Penyelesaian Rekursif: Untuk menyelesaikan persamaan matematika menggunakan binary tree, diperlukan penyelesaian secara rekursif terhadap subtree dari binary tree tersebut. Rekursif dilakukan dengan mengoperasikan left child dan right child dengan operator yang sesuai, lalu hasilnya digunakan sebagai nilai balik untuk level di atasnya.
  • Base Case: Kondisi berhenti (base case) untuk fungsi rekursif ini adalah pada saat pointer berada di leaf node. Karena leaf node tidak memiliki child lagi dan hanya berisi informasi operand, maka nilai balik fungsi rekursifnya adalah informasi node itu sendiri (key).
  • Modul Operator: Untuk mengeksekusi operasi aritmatika string seperti '+', '-', '*', dan '/', digunakan modul bawaan Python yaitu modul operator (misal: operator.add(x, y)).

# IMPLEMENTASI CODE: FUNGSI EVALUASI TREE
import operator

def evaluate(parse_tree):
    # Mapping simbol ke fungsi matematika dari modul operator
    opers = {
        '+': operator.add,
        '-': operator.sub,
        '*': operator.mul,
        '/': operator.truediv
    }

    # Ambil anak kiri dan kanan
    left = parse_tree.getLeftChild()
    right = parse_tree.getRightChild()

    # Jika memiliki anak kiri dan kanan (artinya node adalah Operator)
    if left and right:
        fn = opers[parse_tree.getRootVal()]
        # Lakukan pemanggilan rekursif ke level bawahnya
        return fn(evaluate(left), evaluate(right))

    # BASE CASE: Jika berada di leaf node (artinya node adalah Operand)
    else:
        return parse_tree.getRootVal()
